# docs Instructions

- Mantenha a documentacao curta, especifica e reutilizavel.
- Atualize a documentacao sempre que contratos, fluxos ou limites mudarem.
- Crie uma ADR quando houver mudanca arquitetural relevante.

- Evite duplicacao longa entre README, ADR e prompts.
- Prefira descrever decisoes, limites e interfaces.
- Evite descrever implementacao detalhada de codigo.

- Nao expandir documentacao alem do necessario para o MVP atual.

- README raiz define o contexto geral do projeto.
- docs/product define o comportamento do produto.
- ADRs registram decisoes arquiteturais.
