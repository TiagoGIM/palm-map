# Dev Task Template

## Objetivo

Descreva o resultado esperado de forma concreta.

## Contexto

Explique o motivo da tarefa, o fluxo afetado e os contratos relevantes.

## Escopo

- Liste o que deve ser alterado.
- Cite dominios, apps e docs envolvidos.
- Referenciar contratos existentes em packages/shared-types quando aplicavel
- Manter mudancas pequenas, locais e reversiveis

## Fora de Escopo

- Liste o que nao deve ser implementado.
- Registre integracoes, refactors ou features adiadas.

## Input / Output Esperado

- Input: descrever dados de entrada esperados
- Output: descrever estrutura de saida esperada

## Arquivos Permitidos

- Informe arquivos e pastas que podem ser alterados.

## Arquivos Proibidos

- Informe areas que nao devem ser tocadas.

## Entregaveis

- Codigo
- Documentacao
- Testes ou fixtures, se aplicavel

## Definition of Done

- Comportamento esperado implementado
- Contratos consistentes
- Docs atualizadas
- Sem expandir o escopo do ticket
- Nenhuma regra de negocio foi inferida fora dos contratos definidos

## Validacao

- Como verificar localmente
- Casos minimos de sucesso e erro
- Quais mocks, fixtures ou testes devem existir

## Regras de Execucao

- Leia apenas os arquivos relevantes para a tarefa
- Evite explorar o repositorio inteiro
- Prefira contexto local ao escopo global
